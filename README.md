MedPlus API
MedPlus is an Express.js application that manages pharmacies and medicines. It includes features such as CRUD operations for pharmacies and medicines, user management, and image uploads using Cloudinary. The API is documented with Swagger for easy interaction and testing.

Prerequisites
To run this application locally, you will need:

Node.js (v14 or higher)
npm or yarn
MongoDB (You can use MongoDB Atlas for cloud-based MongoDB)
Cloudinary account
Installation
1. Clone the Repository
Clone the repository to your local machine using the following command:

sh
Copy code
git clone <repository_url>
cd <repository_name>
Replace <repository_url> with the URL of your GitHub repository and <repository_name> with the name of your project directory.

2. Install Dependencies
Navigate to the project directory and install the required dependencies:

sh
Copy code
npm install
# or
yarn install
3. Set Up Environment Variables
Create a .env file in the root directory of the project with the following contents:

env
Copy code
PORT=3000
MONGODB_URI=<your_mongodb_connection_string>
CLOUDINARY_CLOUD_NAME=<your_cloudinary_cloud_name>
CLOUDINARY_API_KEY=<your_cloudinary_api_key>
CLOUDINARY_API_SECRET=<your_cloudinary_api_secret>
Replace the placeholder values with your MongoDB connection string and Cloudinary credentials.

4. Run the Application
Start the application using the following command:

sh
Copy code
npm start
# or
yarn start
The application will be available at http://localhost:3000.

5. Access API Documentation
Head over to http://localhost:3000/api-docs to view the API documentation generated by Swagger.

6. Test API Endpoints
The best way to test the API endpoints is to import the Medlr.postman_collection.json file into Postman. You can download the file from the following link:

Medlr.postman_collection.json

API Endpoints
Pharmacies
Get All Pharmacies

GET /api/v1/pharmacies
Get Pharmacy by ID

GET /api/v1/pharmacies/:id
Create Pharmacy

POST /api/v1/pharmacies
Request Body:
json
Copy code
{
  "name": "Pharmacy Name",
  "address": {
    "city": "City",
    "state": "State",
    "zip_code": "Zip Code",
    "location": "Location",
    "_id": "Pharmacy ID"
  },
  "contactInformation": "Contact Info",
  "phone_number": "Phone Number",
  "email": "Email",
  "license_number": "License Number",
  "owner_name": "Owner Name"
}
Update Pharmacy

PUT /api/v1/pharmacies/:id
Request Body:
json
Copy code
{
  "name": "Updated Pharmacy Name",
  "address": {
    "city": "Updated City",
    "state": "Updated State",
    "zip_code": "Updated Zip Code",
    "location": "Updated Location",
    "_id": "Updated Pharmacy ID"
  },
  "contactInformation": "Updated Contact Info",
  "phone_number": "Updated Phone Number",
  "email": "Updated Email",
  "license_number": "Updated License Number",
  "owner_name": "Updated Owner Name"
}
Delete Pharmacy

DELETE /api/v1/pharmacies/:id

Troubleshooting
MongoDB Connection Issues

Ensure your MongoDB URI is correct and accessible.
Check MongoDB service status.
Cloudinary Upload Issues

Verify Cloudinary credentials in the .env file.
Ensure the image file is included in the form data during the request.
